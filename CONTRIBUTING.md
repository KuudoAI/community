# Contributing to Kuudo

Everything starts here in the community repo. You don't need to write code to contribute — most of the contributions that shape Kuudo are reports, requests, and shared experience.

## Ways to contribute

- 🐛 **Report a bug** — [file it through the guided form](https://github.com/KuudoAI/community/issues/new?template=bug_report.yml). Covers every Kuudo product: the MCP servers, Skills, and Amazon Agent Iris.
- 💡 **Request a feature** — [tell us what you're trying to get done](https://github.com/KuudoAI/community/issues/new?template=feature_request.yml). Requests move across the [public roadmap](https://github.com/orgs/KuudoAI/projects) — 👍 the ones you want prioritized.
- 💬 **Share what you know** — [Discussions](https://github.com/orgs/KuudoAI/discussions) is where setup help, client configuration, and workflow ideas live. Answering someone's question is a contribution.
- 🔧 **Contribute code** — our open-source MCP server for the Amazon Ads API is [`amazon_ads_mcp`](https://github.com/KuudoAI/amazon_ads_mcp); its repo has the specifics on running tests and submitting pull requests.

## A few ground rules

- One issue per problem or request — it keeps the roadmap honest.
- Fill in the form fields; the product and client details are what make reports actionable.
- Search [existing issues](https://github.com/KuudoAI/community/issues) and [discussions](https://github.com/orgs/KuudoAI/discussions) first — a 👍 on an existing thread counts for more than a duplicate.
- Be direct, be kind. We read everything.

## Not sure where something goes?

Open a [discussion](https://github.com/orgs/KuudoAI/discussions) and we'll sort it out together.
